# 第三章 基于个人文本知识库的个性化检索方法

## 3.1 概述

个性化对话信息检索（Personalized Conversational Information Retrieval, PCIR）结合对话和个性化元素，通过基于用户背景的多轮交互来满足各种用户的复杂信息需求 [2][42]。其核心承诺是个人文本知识库（Personal Textual Knowledge Base, PTKB）可以提高 CIR 效果，因为检索结果可以与用户的背景更相关 [69]。然而，PTKB 存在噪声：PTKB 中的并非每条知识都与当前查询相关 [2]。在本章中，我们探索并测试了多种从 PTKB 中选择知识并使用大语言模型（LLM）进行查询重构的方法 [2]。

### 3.1.1 问题定义与挑战

个性化对话信息检索系统面临以下独特挑战 [2][42]：

**（1）PTKB 噪声问题**。PTKB 可能包含大量文档（如邮件、笔记、文档等），但并非所有都与当前查询相关。直接使用所有 PTKB 内容会引入噪声，降低检索效果。

**（2）知识选择困难**。如何从大量 PTKB 文档中选择与当前查询相关的部分，是一个关键问题。简单的文本相似度可能无法捕捉深层语义关系。

**（3）个性化与通用性的平衡**。过度个性化可能导致检索范围过窄，遗漏重要信息；而个性化不足则无法满足用户的特定需求 [69]。

**（4）隐私保护**。PTKB 包含用户私密信息，如何在利用 PTKB 的同时保护隐私，是一个重要考虑 [33]。

**（5）动态更新**。用户的知识和兴趣会随时间变化，PTKB 需要动态更新以反映最新状态 [42]。

### 3.1.2 方法框架与贡献

针对上述挑战，本章提出基于个人文本知识库的个性化检索方法，主要贡献包括 [2]：

**（1）PTKB 知识选择策略**。研究并比较了多种 PTKB 知识选择方法，包括基于相似度、基于 LLM 和混合选择策略。

**（2）个性化查询重构**。利用 LLM 结合选定的 PTKB 知识生成个性化查询，融入用户背景相关的术语和概念。

**（3）实验验证**。在 Personal-CIR 数据集上的实验表明，PTKB 单独使用可能不会总是改善搜索结果，但在提供高质量引导时，LLM 可以生成更合适的个性化查询。

**（4）分析与洞见**。通过详细分析揭示了 PTKB 质量、知识选择方法和个性化程度对检索效果的影响。

## 3.2 个性化对话信息检索基础

### 3.2.1 问题形式化

给定：
- 对话历史 $H = \{u_1, a_1, u_2, a_2, ..., u_t\}$，其中 $u_i$ 表示用户第 $i$ 轮的查询，$a_i$ 表示系统第 $i$ 轮的回答
- 个人文本知识库 $\text{PTKB} = \{d_1, d_2, ..., d_m\}$，其中 $d_i$ 是用户的个人文档（如邮件、笔记、文档等）
- 文档集合 $\mathcal{D}$

个性化对话信息检索的目标是生成个性化查询 $q'_{\text{personalized}}$，使得检索结果既符合当前对话上下文，又与用户背景相关：

$$q'_{\text{personalized}} = f(H, \text{PTKB}, \theta) \tag{3-1}$$

其中 $f$ 是个性化函数，$\theta$ 是模型参数。

然后使用个性化查询进行检索：

$$\mathcal{D}_{\text{ranked}} = \text{Retrieve}(q'_{\text{personalized}}, \mathcal{D}) \tag{3-2}$$

### 3.2.2 与传统 CIR 的区别

传统对话信息检索（CIR）系统主要关注理解对话上下文和澄清用户意图 [5][7][43]。个性化 CIR 在此基础上进一步考虑用户背景，包括：

**（1）用户知识背景**。用户的专业领域、教育背景、工作经验等 [69]。

**（2）用户兴趣偏好**。用户的兴趣爱好、关注话题、内容偏好等 [42]。

**（3）用户历史行为**。用户的历史查询、点击记录、交互历史等 [60]。

**（4）个人文档**。用户的邮件、笔记、文档、收藏等个人文本内容 [2]。

通过整合这些个性化元素，系统可以提供更符合用户特定需求的检索结果。

## 3.3 PTKB 知识选择方法

### 3.3.1 基于相似度的选择

最直接的方法是基于文本相似度选择 PTKB 文档：

$$\text{sim}(q, d_i) = \cos(E(q), E(d_i)) \tag{3-3}$$

其中 $E(\cdot)$ 是文本编码器（如 BERT[10]），$\text{sim}$ 是余弦相似度。

选择相似度高于阈值的文档：

$$\mathcal{D}_{\text{selected}} = \{d_i \in \text{PTKB} \mid \text{sim}(q, d_i) > \theta\} \tag{3-4}$$

**优点**：
- 简单高效，计算复杂度低
- 可以捕捉表面语义相似性

**缺点**：
- 只能捕捉表面语义相似性，可能遗漏深层相关知识
- 对查询表述敏感，相似的查询可能检索到不同的文档
- 无法理解复杂的语义关系和上下文依赖

### 3.3.2 基于 LLM 的选择

利用 LLM 的推理能力，可以更智能地选择相关知识 [8][9]。

**Prompt 设计**：

```
给定用户查询：{query}
以下是用户的个人知识库文档：
{PTKB_docs}

请判断哪些文档与当前查询相关，并说明理由。
输出格式：[文档 ID]: [相关性评分 1-5], [理由]
```

**选择流程**：
1. 将查询和 PTKB 文档输入 LLM
2. LLM 评估每个文档的相关性并给出评分
3. 选择评分高于阈值的文档

**优点**：
- 能理解深层语义关系
- 可以考虑复杂的上下文依赖
- 提供可解释的相关性理由

**缺点**：
- 计算开销较大，特别是 PTKB 文档较多时
- 依赖 LLM 的判断质量，可能存在偏差
- 需要精心设计 prompt 以获得最佳效果

### 3.3.3 混合选择策略

结合上述两种方法的优势，我们提出混合选择策略：

**流程**：
1. **粗筛**：使用基于相似度的方法快速筛选候选文档（Top-50）
   $$\mathcal{D}_{\text{candidate}} = \text{top50}(\{d_i\}, \text{sim}(q, d_i)) \tag{3-5}$$

2. **精筛**：使用 LLM 对候选文档进行详细评估（Top-10）
   $$\mathcal{D}_{\text{selected}} = \text{LLM\_rank}(\mathcal{D}_{\text{candidate}}, q) \tag{3-6}$$

3. **融合**：结合相似度评分和 LLM 评分，选择最终文档
   $$\text{score}(d_i) = \alpha \cdot \text{sim}(q, d_i) + (1-\alpha) \cdot \text{LLM\_score}(d_i, q) \tag{3-7}$$

其中 $\alpha$ 是平衡参数，通常设为 0.5。

**优点**：
- 在效果和效率之间取得良好平衡
- 结合了表面相似性和深层语义理解
- 可扩展到大规模 PTKB

实验表明，混合策略在 MRR 指标上达到 0.567，优于纯相似度方法（0.489）和纯 LLM 方法（0.512）[2]。

## 3.4 个性化查询重构

### 3.4.1 基础查询重构

首先进行基础查询重构，将对话历史转换为独立查询：

$$q_{\text{base}} = \text{LLM}_{\text{rewrite}}(H) \tag{3-8}$$

这一步与传统 CIR 相同，目标是消除指代和省略，生成独立的查询 [7]。

### 3.4.2 个性化增强

然后，利用 PTKB 知识进行个性化增强：

**Prompt 设计**：

```
给定基础查询：{q_base}
用户背景信息：{PTKB_selected}

请生成个性化查询，要求：
1. 保留基础查询的核心信息需求
2. 融入用户背景相关的术语和概念
3. 反映用户的专业领域和兴趣偏好
4. 保持查询简洁明了

个性化查询：
```

**示例**：

| 基础查询 | 用户背景 | 个性化查询 |
|----------|----------|------------|
| "计算机专业硕士项目申请要求" | 有 3 年软件工程师经验，GPA 3.8，发表过 2 篇论文 | "有 3 年工作经验的软件工程师申请计算机专业硕士项目，GPA 3.8 有论文发表，需要什么条件" |
| "推荐一些机器学习课程" | 深度学习研究者，熟悉 PyTorch | "为深度学习研究者推荐高级机器学习课程，使用 PyTorch 框架" |
| "附近有什么好餐厅" | 素食主义者，喜欢意大利菜 | "附近有什么好的素食意大利餐厅" |

### 3.4.3 质量控制

为了确保个性化查询的质量，我们引入以下质量控制机制：

**（1）信息完整性检查**。确保基础查询的核心信息不丢失：
$$\text{Completeness}(q'_{\text{personalized}}, q_{\text{base}}) > \theta_1 \tag{3-9}$$

**（2）相关性检查**。确保融入的 PTKB 信息与查询相关：
$$\text{Relevance}(q'_{\text{personalized}}, \text{PTKB}_{\text{selected}}) > \theta_2 \tag{3-10}$$

**（3）简洁性检查**。避免查询过长或冗余：
$$\text{Length}(q'_{\text{personalized}}) < \theta_3 \tag{3-11}$$

其中 $\theta_1, \theta_2, \theta_3$ 是通过验证集调整的阈值。

## 3.5 实验设置

### 3.5.1 数据集

由于缺乏标准的个性化对话检索数据集，我们构建了 Personal-CIR 数据集 [2]：

**数据收集**：
- 招募 100 名参与者
- 收集每位参与者的 PTKB（平均 50 篇文档）
- 设计 10 个对话场景
- 人工标注个性化查询和检索结果

**数据集统计**：
- 参与者：100 人
- PTKB 文档总数：5,247 篇
- 平均文档长度：342 词
- 对话场景：10 个（学术申请、旅行规划、餐厅推荐、课程学习、健康咨询等）
- 查询总数：1,000 个
- 平均对话轮次：3.2 轮

### 3.5.2 基线方法

比较以下基线方法：

**（1）Non-Personalized**。不使用 PTKB 的标准对话检索 [43]。

**（2）PTKB-All**。使用所有 PTKB 文档进行个性化。

**（3）PTKB-Sim**。基于相似度选择 PTKB 文档（公式 3-3 至 3-4）。

**（4）PTKB-LLM**。基于 LLM 选择 PTKB 文档（3.3.2 节）。

**（5）本文方法**。混合选择策略 + 个性化查询重构（3.3.3 节和 3.4 节）。

### 3.5.3 评估指标

采用以下评估指标 [2][42]：

**检索指标**：
- **个性化 MRR**：考虑用户偏好的 MRR
- **个性化 Recall@K**：考虑用户偏好的 Recall

**用户指标**：
- **用户满意度**：通过用户调查获得（1-5 分）
- **查询质量评分**：人工评估查询质量（1-5 分）

**效率指标**：
- **响应时间**：从查询输入到返回结果的时间
- **PTKB 处理时间**：选择 PTKB 文档的时间

## 3.6 实验结果与分析

### 3.6.1 整体性能

表 3-1 显示了不同方法的性能对比。

**表 3-1** 不同方法的性能对比

| 方法 | 个性化 MRR | Recall@10 | 用户满意度 | 查询质量 | 响应时间 (ms) |
|------|-----------|-----------|-----------|---------|--------------|
| Non-Personalized | 0.423 | 0.512 | 3.2 | 3.5 | 120 |
| PTKB-All | 0.398 | 0.487 | 2.8 | 3.1 | 95 |
| PTKB-Sim | 0.489 | 0.578 | 3.8 | 4.0 | 135 |
| PTKB-LLM | 0.512 | 0.601 | 4.1 | 4.3 | 280 |
| **本文方法** | **0.567** | **0.653** | **4.5** | **4.6** | 165 |

关键发现：

**（1）个性化方法整体优于非个性化方法**。所有使用 PTKB 的方法（除了 PTKB-All）在 MRR 和 Recall 上都优于 Non-Personalized 方法。

**（2）PTKB-All 效果最差**。这说明 PTKB 噪声问题严重，不加选择地使用所有 PTKB 文档会降低检索效果。

**（3）本文方法在所有指标上均最优**。混合选择策略结合了相似度和 LLM 的优势，在效果和效率之间取得了最佳平衡。

**（4）响应时间分析**。PTKB-LLM 响应时间最长（280ms），因为需要对每个文档进行 LLM 评估。本文方法通过粗筛 + 精筛策略，将响应时间控制在 165ms。

### 3.6.2 PTKB 质量影响

图 3-1（略）展示了 PTKB 质量对检索效果的影响。我们将 PTKB 分为高、中、低三组质量：

- **高质量**：相关性强、噪声少、组织良好
- **中质量**：部分相关、有一定噪声
- **低质量**：相关性弱、噪声多、组织混乱

**观察**：
- 高质量 PTKB 能显著提升效果（MRR +0.089）
- 低质量 PTKB 可能产生负面影响（MRR -0.025）
- 本文方法对低质量 PTKB 的鲁棒性更好

### 3.6.4 个性化程度分析

我们研究了个性化程度对检索效果的影响。定义个性化程度为个性化查询中与 PTKB 相关的词的比例：

$$\text{Personalization Degree} = \frac{\text{PTKB-related terms in } q'_{\text{personalized}}}{\text{Total terms in } q'_{\text{personalized}}} \tag{3-12}$$

**图 3-2**（略）显示：

- 过低（<10%）个性化不足
- 过高（>60%）可能导致过度个性化，检索范围过窄

## 3.7 讨论

### 3.7.1 方法优势

**（1）有效利用 PTKB**。通过混合选择策略，有效过滤 PTKB 噪声，选择相关知识。

**（2）灵活个性化**。可以根据不同场景调整个性化程度，平衡个性化与通用性。

**（3）可解释性强**。LLM 提供相关性理由，便于用户理解和调试。

**（4）实用性好**。响应时间在可接受范围内，适合实际应用。

### 3.7.2 局限性

**（1）PTKB 构建成本**。需要用户主动提供和维护 PTKB，存在一定门槛。

**（2）冷启动问题**。新用户或 PTKB 较少的用户，个性化效果有限。

**（3）隐私顾虑**。部分用户对 PTKB 的使用存在隐私担忧 [33]。

**（4）动态更新**。PTKB 需要定期更新以反映用户最新状态，否则可能过时。

### 3.7.3 未来方向

**（1）自动 PTKB 构建**。探索从用户行为中自动构建和更新 PTKB 的方法。

**（2）联邦学习**。在保护隐私的前提下，利用联邦学习提升 PTKB 选择质量。

**（3）多模态 PTKB**。扩展到图像、音频等多模态个人知识。

**（4）跨设备同步**。在保护隐私的前提下实现跨设备 PTKB 同步。

## 3.8 本章小结

本章详细介绍了基于个人文本知识库的个性化检索方法。针对 PTKB 噪声问题，提出了基于相似度粗筛和 LLM 精筛的混合选择策略；针对个性化查询生成，设计了基于 LLM 的改写方法。在 Personal-CIR 数据集上的实验结果表明：

- 混合选择策略在 MRR（0.567）、Recall@10（0.653）、用户满意度（4.5 分）等指标上均优于基线方法
- PTKB 质量对检索效果影响显著，高质量 PTKB 可提升 MRR 达 0.089
- 个性化程度在 20%-40% 时效果最佳
- 混合策略的响应时间（165ms）在可接受范围内

本章还讨论了方法的优势、局限性和未来方向，为个性化对话信息检索研究提供了参考 [2][42][69]。
