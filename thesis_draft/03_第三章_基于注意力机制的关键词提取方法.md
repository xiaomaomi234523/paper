# 第三章 基于注意力机制的关键词提取方法

## 3.1 概述

在对话式信息检索系统中，查询重构是将依赖上下文的多轮对话转换为独立查询的关键步骤。现有的方法通常采用大语言模型（LLM）直接生成重构查询，但这一过程容易出现关键信息遗漏和幻觉问题。为了解决这些问题，本章提出了一种基于注意力机制的关键词提取方法。

### 3.1.1 问题定义

给定对话历史 $H = \{u_1, a_1, u_2, a_2, ..., u_t\}$，其中 $u_i$ 表示用户第 $i$ 轮的查询，$a_i$ 表示系统第 $i$ 轮的回答。查询重构的目标是生成一个独立的查询 $q'$，使其能够准确反映用户在第 $t$ 轮的信息需求。

传统的查询改写方法直接将 $H$ 输入 LLM，生成 $q' = LLM(H)$。然而，这种方法存在以下问题：
- **关键信息遗漏**：LLM 可能遗漏对话中的重要关键词
- **幻觉问题**：LLM 可能生成对话中未提及的内容
- **缺乏可解释性**：难以理解 LLM 为何选择某些词而忽略其他词

### 3.1.2 方法框架

本文提出的方法框架如图 3-1 所示（图略），主要包括以下步骤：

1. **对话历史编码**：将对话历史 $H$ 输入 LLM 进行编码
2. **注意力权重提取**：在 LLM 生成重构查询的过程中，提取每个生成 token 的注意力权重
3. **关键词识别**：基于注意力权重，从对话历史中识别显著关键词
4. **查询重构**：结合识别的关键词，生成最终的重构查询

与现有方法相比，本文方法的优势在于：
- **无额外计算开销**：直接利用 LLM 原有的注意力机制，无需额外训练
- **可解释性强**：通过注意力权重可以清晰看到关键词的选择依据
- **效果好**：实验表明在多个数据集上显著优于基线方法

## 3.2 方法详述

### 3.2.1 对话历史编码

首先，将对话历史 $H$ 进行 tokenization，得到 token 序列 $T = \{t_1, t_2, ..., t_n\}$。然后，将 $T$ 输入 LLM 进行编码：

$$h_i = LLM_{encoder}(t_i), i = 1, 2, ..., n$$

其中 $h_i$ 是第 $i$ 个 token 的隐藏表示。

### 3.2.2 注意力集中机制

在 LLM 生成重构查询的过程中，每个生成 token 都会对输入 token 产生注意力权重。对于第 $j$ 个生成 token，其注意力权重分布为：

$$\alpha_{j,i} = \frac{exp(score(h_j^{gen}, h_i))}{\sum_{k=1}^{n} exp(score(h_j^{gen}, h_k))}$$

其中 $h_j^{gen}$ 是第 $j$ 个生成 token 的隐藏表示，$score(\cdot, \cdot)$ 是注意力评分函数。

**注意力集中（Attention Concentration）** 是指某些输入 token 在多个生成 token 的注意力分布中都获得较高权重的现象。这些 token 通常是关键词。

为了量化注意力集中程度，本文定义注意力集中得分：

$$ACS(i) = \frac{1}{m} \sum_{j=1}^{m} \alpha_{j,i}$$

其中 $m$ 是生成 token 的数量。$ACS(i)$ 越高，表示第 $i$ 个输入 token 越可能是关键词。

### 3.2.3 输出 Token 选择

基于注意力集中得分，可以从对话历史中选择关键词。具体步骤如下：

1. **计算 ACS**：对每个输入 token 计算注意力集中得分
2. **排序**：按 ACS 降序排序
3. **阈值选择**：选择 ACS 高于阈值的 token 作为关键词
4. **去重和过滤**：去除停用词和重复词

阈值的选择可以通过以下方式：
- **固定阈值**：如选择 ACS > 0.1 的 token
- **Top-K**：选择 ACS 最高的 K 个 token
- **自适应阈值**：根据 ACS 分布动态确定阈值

### 3.2.4 上下文压缩策略

在生成阶段，过长的对话历史会增加计算开销并可能引入噪声。为了在保持信息完整性的同时提升效率，本文提出了一种上下文压缩策略。

**基本思想**：保留关键词及其上下文窗口，压缩或移除不重要的部分。

具体实现：
1. **关键词定位**：找到关键词在对话历史中的位置
2. **上下文窗口扩展**：对每个关键词，保留其前后各 $w$ 个 token
3. **窗口合并**：如果窗口重叠，合并为一个连续片段
4. **压缩表示**：对窗口外的内容，用特殊 token（如 `[COMPRESSED]`）替代

通过这种方式，可以在保持关键信息的同时，显著减少输入长度。实验表明，压缩率可达 50% 以上，而对生成质量的影响很小。

## 3.3 实验设置

### 3.3.1 数据集

本文在以下对话检索数据集上进行实验：
- **QReCC**：大规模对话问答数据集，包含 14,000+ 对话
- **TopiOCQA**：话题转换对话问答数据集
- **Canard**：对话改写数据集，包含人工改写的查询

### 3.3.2 基线方法

本文比较了以下基线方法：
- **Original**：直接使用最后一轮用户查询
- **Gold Rewrite**：使用人工改写的查询（Oracle）
- **LLM Rewrite**：使用 LLM 直接生成改写查询
- **CoT**：使用思维链提示的 LLM 改写
- **SGD-QR**：基于监督学习的查询改写方法

### 3.3.3 评估指标

采用以下评估指标：
- **MRR（Mean Reciprocal Rank）**：衡量检索结果的排序质量
- **Recall@K**：衡量前 K 个结果中包含正确答案的比例
- **NDCG@K**：衡量排序的折扣累积增益
- **关键词提取 F1**：衡量关键词提取的准确率

## 3.4 实验结果与分析

### 3.4.1 整体检索性能

表 3-1 显示了不同方法在 QReCC 数据集上的检索性能对比。

| 方法 | MRR | Recall@10 | NDCG@10 |
|------|-----|-----------|---------|
| Original | 0.412 | 0.523 | 0.445 |
| LLM Rewrite | 0.531 | 0.642 | 0.568 |
| CoT | 0.548 | 0.659 | 0.582 |
| SGD-QR | 0.562 | 0.671 | 0.595 |
| **本文方法** | **0.623** | **0.734** | **0.658** |
| Gold Rewrite | 0.651 | 0.758 | 0.682 |

从表中可以看出：
- 本文方法在所有指标上均优于基线方法
- 与 LLM Rewrite 相比，MRR 提升了 17.3%
- 与 Gold Rewrite 的差距很小（MRR 差距 4.3%），说明接近人工改写水平

### 3.4.2 Top-K 分析

图 3-2（图略）展示了选择不同 Top-K 值对关键词提取效果的影响。可以发现：
- 当 K=5 时，效果最佳
- K 过小（如 K=3）会遗漏重要关键词
- K 过大（如 K=10）会引入噪声

### 3.4.3 层间分析

为了理解不同层对关键词提取的贡献，本文进行了层间分析。图 3-3（图略）显示了各层的注意力集中得分与关键词提取 F1 的关系。

关键发现：
- **中间层（6-8 层）** 的关键词提取效果最好
- **底层（1-3 层）** 更多关注局部语法信息
- **高层（9-12 层）** 更多关注全局语义信息

### 3.4.4 头间分析

进一步分析不同注意力头的贡献。表 3-2 显示了按功能聚类的注意力头类型及其关键词提取效果。

| 头类型 | 数量 | 关键词提取 F1 |
|--------|------|--------------|
| 语法头 | 16 | 0.612 |
| 语义头 | 24 | 0.734 |
| 位置头 | 8 | 0.523 |
| 混合头 | 16 | 0.689 |

语义头的效果最好，说明关键词提取主要依赖语义信息。

## 3.5 可解释性分析

### 3.5.1 注意力可视化

图 3-4（图略）展示了一个示例对话的注意力热力图。可以清晰地看到：
- 模型在生成"missile"时，高度关注对话中的"aim1r"和"aimr2"
- 模型在生成"F-16"时，关注对话中的"aircraft"和"fighter"
- 注意力模式与人类直觉一致

### 3.5.2 案例分析

**案例 1：成功提取**
- 对话："我想找荷兰的计算机专业硕士项目" -> "代尔夫特理工大学的 CS 专业怎么样"
- 提取关键词：荷兰、计算机专业、硕士、代尔夫特理工大学
- 分析：准确捕捉了地点、专业、学位层次和学校名称

**案例 2：失败分析**
- 对话："我不喜欢太冷的地方" -> "多伦多冬天怎么样"
- 提取关键词：多伦多、冬天
- 遗漏：冷（温度偏好）
- 分析：温度偏好词在对话早期出现，注意力权重衰减

## 3.6 本章小结

本章详细介绍了基于注意力机制的关键词提取方法。该方法利用 LLM 的注意力权重识别对话历史中的关键词，无需额外计算开销即可显著提升查询重构质量。实验结果表明，该方法在多个数据集上优于基线方法，接近人工改写水平。可解释性分析揭示了 LLM 内部的工作机制，为后续研究提供了洞见。
