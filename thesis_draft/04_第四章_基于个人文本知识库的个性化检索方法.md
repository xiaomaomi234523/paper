# 第四章 基于个人文本知识库的个性化检索方法

## 4.1 概述

随着信息检索系统的广泛应用，用户对个性化服务的需求日益增长。个性化对话信息检索（Personalized Conversational Information Retrieval, PCIR）旨在结合对话历史和用户个人背景，满足用户复杂的信息需求。本章研究如何利用个人文本知识库（Personal Textual Knowledge Base, PTKB）进行个性化检索。

### 4.1.1 问题定义

给定：
- 对话历史 $H = \{u_1, a_1, ..., u_t\}$
- 个人文本知识库 $PTKB = \{d_1, d_2, ..., d_m\}$，其中 $d_i$ 是用户的个人文档（如邮件、笔记、文档等）

目标是生成个性化查询 $q'_{personalized}$，使得检索结果既符合当前对话上下文，又与用户背景相关。

### 4.1.2 挑战

利用 PTKB 进行个性化检索面临以下挑战：

**（1）PTKB 噪声问题**。PTKB 中的文档并非都与当前查询相关。直接使用所有 PTKB 内容会引入噪声，降低检索效果。

**（2）知识选择困难**。如何从大量 PTKB 文档中选择与当前查询相关的部分，是一个关键问题。

**（3）个性化与通用性的平衡**。过度个性化可能导致检索范围过窄，遗漏重要信息。

**（4）隐私保护**。PTKB 包含用户私密信息，如何在利用 PTKB 的同时保护隐私，是一个重要考虑。

### 4.1.3 方法框架

本文提出的方法框架如图 4-1 所示（图略），主要包括以下步骤：

1. **PTKB 预处理**：对 PTKB 文档进行分块和索引
2. **相关知识检索**：从 PTKB 中检索与当前查询相关的文档
3. **个性化查询改写**：利用 LLM 结合 PTKB 知识生成个性化查询
4. **检索与排序**：使用个性化查询进行检索，并对结果进行个性化排序

## 4.2 PTKB 知识选择方法

### 4.2.1 基于相似度的选择

最直接的方法是基于文本相似度选择 PTKB 文档：

$$sim(q, d_i) = \cos(E(q), E(d_i))$$

其中 $E(\cdot)$ 是文本编码器（如 BERT），$sim$ 是余弦相似度。

选择相似度高于阈值的文档：
$$D_{selected} = \{d_i \in PTKB | sim(q, d_i) > \theta\}$$

**优点**：简单高效
**缺点**：只能捕捉表面语义相似性，可能遗漏深层相关知识

### 4.2.2 基于 LLM 的选择

利用 LLM 的推理能力，可以更智能地选择相关知识：

**Prompt 设计**：
```
给定用户查询：{query}
以下是用户的个人知识库文档：
{PTKB_docs}

请判断哪些文档与当前查询相关，并说明理由。
输出格式：[文档 ID]: [相关性评分 1-5], [理由]
```

**优点**：能理解深层语义关系
**缺点**：计算开销较大

### 4.2.3 混合选择策略

结合上述两种方法的优势，本文提出混合选择策略：

1. **粗筛**：使用基于相似度的方法快速筛选候选文档（Top-50）
2. **精筛**：使用 LLM 对候选文档进行详细评估（Top-10）
3. **融合**：结合相似度评分和 LLM 评分，选择最终文档

实验表明，混合策略在效果和效率之间取得了良好平衡。

## 4.3 个性化查询改写

### 4.3.1 基础查询改写

首先进行基础查询改写，将对话历史转换为独立查询：

$$q_{base} = LLM_{rewrite}(H)$$

### 4.3.2 个性化增强

然后，利用 PTKB 知识进行个性化增强：

**Prompt 设计**：
```
给定基础查询：{q_base}
用户背景信息：{PTKB_selected}

请生成个性化查询，要求：
1. 保留基础查询的核心信息需求
2. 融入用户背景相关的术语和概念
3. 反映用户的专业领域和兴趣偏好

个性化查询：
```

**示例**：
- 基础查询："计算机专业硕士项目申请要求"
- 用户背景：有 3 年软件工程师经验，GPA 3.8，发表过 2 篇论文
- 个性化查询："有 3 年工作经验的软件工程师申请计算机专业硕士项目，GPA 3.8 有论文发表，需要什么条件"

### 4.3.3 质量控制

为了确保个性化查询的质量，本文引入以下质量控制机制：

**（1）信息完整性检查**：确保基础查询的核心信息不丢失
**（2）相关性检查**：确保融入的 PTKB 信息与查询相关
**（3）简洁性检查**：避免查询过长或冗余

## 4.4 实验设置

### 4.4.1 数据集

由于缺乏标准的个性化对话检索数据集，本文构建了以下数据集：

**Personal-CIR**：
- 招募 100 名参与者
- 收集每位参与者的 PTKB（平均 50 篇文档）
- 设计 10 个对话场景
- 人工标注个性化查询和检索结果

### 4.4.2 基线方法

比较以下基线方法：
- **Non-Personalized**：不使用 PTKB 的标准对话检索
- **PTKB-All**：使用所有 PTKB 文档
- **PTKB-Sim**：基于相似度选择 PTKB 文档
- **PTKB-LLM**：基于 LLM 选择 PTKB 文档
- **本文方法**：混合选择策略 + 个性化查询改写

### 4.4.3 评估指标

采用以下评估指标：
- **个性化 MRR**：考虑用户偏好的 MRR
- **个性化 Recall@K**：考虑用户偏好的 Recall
- **用户满意度**：通过用户调查获得（1-5 分）
- **查询质量评分**：人工评估查询质量（1-5 分）

## 4.5 实验结果与分析

### 4.5.1 整体性能

表 4-1 显示了不同方法的性能对比。

| 方法 | 个性化 MRR | Recall@10 | 用户满意度 | 查询质量 |
|------|-----------|-----------|-----------|---------|
| Non-Personalized | 0.423 | 0.512 | 3.2 | 3.5 |
| PTKB-All | 0.398 | 0.487 | 2.8 | 3.1 |
| PTKB-Sim | 0.489 | 0.578 | 3.8 | 4.0 |
| PTKB-LLM | 0.512 | 0.601 | 4.1 | 4.3 |
| **本文方法** | **0.567** | **0.653** | **4.5** | **4.6** |

关键发现：
- 个性化方法整体优于非个性化方法
- PTKB-All 效果最差，说明 PTKB 噪声问题严重
- 本文方法在所有指标上均最优

### 4.5.2 PTKB 质量影响

图 4-2（图略）展示了 PTKB 质量对检索效果的影响。可以发现：
- 高质量 PTKB（相关性强、噪声少）能显著提升效果
- 低质量 PTKB 可能产生负面影响
- 本文方法对低质量 PTKB 的鲁棒性更好

### 4.5.3 案例分析

**案例 1：成功个性化**
- 用户：计算机科学教授，研究方向为机器学习
- 查询："最近有什么新进展"
- 非个性化查询："机器学习最新研究进展"
- 个性化查询："2024 年机器学习顶会（ICML/NeurIPS/ICLR）的最新研究进展，特别是大语言模型和强化学习方向"
- 分析：融入了用户的研究领域和关注的会议

**案例 2：个性化失败**
- 用户：有多个兴趣领域
- 查询："推荐一些课程"
- 个性化查询：只推荐了计算机课程（基于 PTKB 中的主要背景）
- 问题：用户实际想了解摄影课程
- 分析：PTKB 未能反映用户的多元兴趣

## 4.6 讨论

### 4.6.1 PTKB 构建策略

基于实验结果，本文提出以下 PTKB 构建建议：
- **定期更新**：保持 PTKB 的时效性
- **主动筛选**：移除过时或不相关的内容
- **结构化组织**：按主题或时间组织文档
- **元数据标注**：添加时间、主题等元数据

### 4.6.2 隐私保护

在利用 PTKB 时，需要考虑隐私保护：
- **本地处理**：PTKB 存储在本地，不上传云端
- **差分隐私**：在查询中添加噪声保护隐私
- **用户控制**：允许用户选择哪些 PTKB 文档可用
- **透明性**：告知用户 PTKB 如何被使用

### 4.6.3 未来方向

个性化检索的未来研究方向包括：
- **多模态 PTKB**：整合文本、图像、音频等多种模态
- **动态更新**：实时更新 PTKB 和用户模型
- **跨设备同步**：在保护隐私的前提下实现跨设备 PTKB 同步
- **联邦学习**：在不共享原始数据的情况下训练个性化模型

## 4.7 本章小结

本章详细介绍了基于个人文本知识库的个性化检索方法。针对 PTKB 噪声问题，提出了混合选择策略；针对个性化查询生成，设计了基于 LLM 的改写方法。实验结果表明，本文方法在多个指标上优于基线方法，用户满意度达到 4.5 分（5 分制）。本章还讨论了 PTKB 构建策略、隐私保护和未来研究方向。
