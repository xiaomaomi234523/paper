# 第四章 基于注意力机制的关键词提取方法

## 4.1 概述

对话式检索系统通常采用大语言模型（LLM）对历史对话进行重构以理解查询 [1][5][7]。然而，这种方法经常出现关键信息遗漏和幻觉问题，导致检索失败 [2][3]。在本章中，我们提出了一种新颖的对话式检索增强生成（RAG）框架，利用 LLM 在重构 token 生成过程中的注意力机制，从对话历史中识别和提取显著关键词 [1]。

### 4.1.1 问题定义与挑战

在对话式检索增强生成（Conversational RAG）系统中，给定多轮对话历史 $H = \{(q_1, r_1), (q_2, r_2), ..., (q_{n-1}, r_{n-1})\}$ 和当前查询 $q_n$，系统的目标是生成准确的回答 $a_n$。传统方法通常采用以下步骤：

1. **查询重构**：使用 LLM 将当前查询重构为独立形式 $q_n' = \text{LLM}(q_n|H)$
2. **文档检索**：使用重构查询检索相关文档 $D = \text{Retrieve}(q_n')$
3. **答案生成**：基于检索文档生成回答 $a_n = \text{Generate}(q_n, D)$

然而，这一流程面临以下关键挑战 [1][2]：

**（1）关键词遗漏问题**。LLM 在生成重构查询时，可能无意中遗漏对话历史中的重要术语。

**（2）幻觉问题**。LLM 可能生成看似合理但与对话历史不符的内容 [34][35][36]。

**（3）长上下文处理**。对话历史可能很长，超出 LLM 的上下文窗口限制 [30]。

**（4）检索与生成的协同**。现有 RAG 系统通常将检索和生成视为独立阶段 [3][37]。

### 4.1.2 方法框架与贡献

针对上述挑战，本章提出基于注意力机制的关键词提取方法，主要贡献包括 [1]：

**（1）注意力驱动的关键词提取**。利用 LLM 在生成重构查询 token 时的注意力分布，识别对话历史中的显著关键词。

**（2）上下文压缩策略**。基于注意力重要性分数，保留包含高重要性关键词的对话轮次。

**（3）混合检索策略**。结合重构查询和提取的关键词，缓解关键词遗漏问题。

**（4）可解释性分析**。系统分析不同 Transformer 层和注意力头对关键词选择和信息检索任务的贡献。

**（5）实验验证**。在 CORAL 对话检索基准上的实验表明，该方法显著优于传统重构方法。

## 4.2 基于注意力的关键词提取方法

### 4.2.1 整体架构

给定多轮对话历史 $H$ 和当前查询 $q_n$，系统执行以下操作：

1. **使用 LLM 进行查询重构**：生成独立查询 $q_n'$
2. **基于注意力的关键词提取**：通过分析重构过程中的注意力模式，从对话历史中提取关键词
3. **混合检索**：结合重构查询和提取的关键词进行检索

### 4.2.2 基于 LLM 的查询重构

查询重构是对话检索中的关键步骤，目标是将依赖上下文的查询转换为独立形式 [5][7]。给定对话历史 $H$ 和当前查询 $q_n$，重构查询 $q_n'$ 生成过程为：

$$q_n' = \text{LLM}(q_n | H) = \{t_1, t_2, ..., t_m\}$$

其中 $t_i$ 是生成的第 $i$ 个 token。

### 4.2.3 注意力权重提取

在生成每个 token $t_i$ 时，Transformer 的注意力机制计算查询 token 对输入 token 的注意力权重：

$$A_{ij}^{(l,h)} = \text{Attention}(t_i, H_j)^{(l,h)}$$

其中 $(l,h)$ 表示第 $l$ 层第 $h$ 个注意力头，$H_j$ 是对话历史中的第 $j$ 个 token。

#### 4.2.3.1 注意力聚合策略

我们探索三种聚合策略：

**（1）层平均（Layer-Averaged）**：
$$\bar{A}_{ij} = \frac{1}{L \times H} \sum_{l=1}^{L} \sum_{h=1}^{H} A_{ij}^{(l,h)}$$

**（2）专家头选择（Expert Head Selection）**：
$$\bar{A}_{ij} = \frac{1}{|\mathcal{E}|} \sum_{(l,h) \in \mathcal{E}} A_{ij}^{(l,h)}$$

**（3）专家头集成（Expert Head Ensemble）**：
保留每个 expert head 的独立权重，在检索阶段进行集成。

#### 4.2.3.2 Top-K 关键词选择

基于聚合后的注意力权重，计算每个输入 token 的重要性分数：

$$s_j = \sum_{i=1}^{m} \bar{A}_{ij}$$

选择分数最高的 K 个 token 作为关键词。

#### 4.2.3.3 基于注意力熵的输出 Token 选择

并非所有生成的 token 都适合用于关键词提取。我们使用注意力熵衡量 token 的信息量：

$$H(t_i) = -\sum_{j} A_{ij} \log A_{ij}$$

过滤掉低熵的停用词和高熵的无意义词。

## 4.3 混合检索策略

### 4.3.1 检索方法

结合重构查询和提取的关键词进行检索：

**（1）BM25**。传统的词频 - 逆文档频率方法 [45]。

**（2）稠密检索**。使用 DPR 等稠密检索器 [46]。

**（3）混合检索**。结合 BM25 和稠密检索的优势。

### 4.3.2 关键词融合策略

将提取的关键词融入检索过程：

$$\text{score}(d) = \alpha \cdot \text{score}_{\text{query}}(d) + (1-\alpha) \cdot \text{score}_{\text{keywords}}(d)$$

其中 $\alpha$ 通过验证集调整。

## 4.4 RAG 生成与上下文压缩

### 4.4.1 生成的上下文压缩

当对话历史过长时，需要进行压缩以适应 LLM 上下文窗口：

**（1）基于轮次的压缩**。保留包含高重要性关键词的对话轮次。

**（2）基于注意力的压缩**。根据注意力权重保留重要内容。

压缩率定义为：
$$\text{Compression Rate} = \frac{\text{Original Length} - \text{Compressed Length}}{\text{Original Length}}$$

### 4.4.2 可解释性分析

通过分析注意力权重，可以解释关键词选择的依据，提高系统透明度 [32]。

### 4.4.3 计算效率

基础方法利用 LLM 已有的注意力权重，无需额外推理成本。

## 4.5 基于学习的动态注意力头加权方法

### 4.5.1 引言

在 4.2 节中，我们提出了 Expert Head Ensemble 方法，通过简单平均多个 expert heads 的注意力权重来提取关键词。虽然这种方法相比单一 head 有显著提升，但简单平均策略存在以下局限性 [1]：

**（1）忽略了上下文差异**。不同的 query 可能需要依赖不同的 attention heads。

**（2）忽略了 token 差异**。生成序列中不同位置的 token 对关键词提取的贡献不同。

**（3）缺乏自适应性**。无法根据具体任务动态调整各 head 的重要性。

为了解决这些问题，本节提出了一种**基于学习的动态注意力头加权方法**（Learned Dynamic Head Weighting, LDHW），通过训练一个轻量级的权重预测器，为每个 (output token, attention head) 对学习最优的权重分配策略。

### 4.5.2 方法设计

#### 4.5.2.1 问题形式化

LDHW 方法将关键词分数计算扩展为：

$$s_j = \sum_{(l,h) \in \mathcal{E}} \sum_{i=1}^{m} w_{i,h} \cdot A_{ij}^{(l,h)}$$

其中 $w_{i,h}$ 是为第 $i$ 个输出 token 和第 $h$ 个 expert head 学习的动态权重。

#### 4.5.2.2 权重预测器架构

**模型结构**包含三个主要模块：

**（1）Token 特征编码器**：$\mathbf{F}_{token} = \phi_{token}(\mathbf{H}_{out})$

**（2）Head 特征编码器**：$\mathbf{F}_{head}^{(l,h)} = \phi_{head}(\text{stat}(\mathbf{A}^{(l,h)}), l, h)$

**（3）Token-Head 交互层**：$\mathbf{F}_{interact} = \text{MHA}(\mathbf{F}_{token}, \mathbf{F}_{head}, \mathbf{F}_{head})$

**（4）权重预测头**：$w_{i,h} = \text{softmax}_h(g([\mathbf{F}_{token}^i; \mathbf{F}_{interact}^i; \mathbf{F}_{head}^h]))$

#### 4.5.2.3 训练目标

多目标损失函数：

$$\mathcal{L}_{total} = \lambda_1 \mathcal{L}_{match} + \lambda_2 \mathcal{L}_{rank} + \lambda_3 \mathcal{L}_{reg}$$

### 4.5.3 实验结果

#### 4.5.3.1 主要结果

LDHW 在三个数据集上均优于基线方法：

| 方法 | QReCC R@5 | TopiOCQA R@5 | Doc2Dial R@5 |
|------|-----------|--------------|--------------|
| Expert Head Ensemble | 52.7 | 49.1 | 53.4 |
| **LDHW (Ours)** | **56.8** | **52.9** | **57.2** |

相比 Expert Head Ensemble，Recall@5 平均提升**4.1 个百分点**。

#### 4.5.3.2 消融实验

- 移除 Token-Head 交互层：性能下降 1.9%
- 移除 Head 统计特征：性能下降 1.6%

#### 4.5.3.3 效率分析

| 方法 | 额外参数量 | 推理时间增加 |
|------|-----------|------------|
| Expert Head Ensemble | 0 | 0 ms |
| **LDHW** | **1.2M** | **5 ms** |

### 4.5.4 本节小结

LDHW 为对话检索中的关键词提取提供了一个更精细、更自适应的解决方案。

## 4.8 讨论

### 4.8.1 方法优势

**（1）零额外计算开销**。基础方法利用 LLM 已有的注意力权重。

**（2）可解释性强**。通过注意力权重可以清晰看到关键词的选择依据 [32]。

**（3）通用性好**。适用于任何基于 Transformer 的 LLM。

**（4）LDHW 自适应性强**。学习的权重能根据 query 类型动态调整策略。

### 4.8.2 局限性

**（1）依赖 LLM 注意力质量**。如果 LLM 的注意力模式不可靠，效果会受影响。

**（2）停用词过滤**。需要预定义停用词表。

**（3）LDHW 需要训练数据**。相比 zero-cost 的简单平均，需要标注数据。

### 4.8.3 未来方向

**（1）动态专家头选择**。根据输入动态选择专家头。

**（2）跨层注意力融合**。探索更复杂的跨层融合策略。

**（3）多模态扩展**。扩展到图像 - 文本、视频 - 文本等场景。

**（4）元学习**。使用元学习实现快速领域适应。

## 4.9 本章小结

本章详细介绍了基于注意力机制的关键词提取方法，包括基础方法和高级的 LDHW 方法。

**基础方法**：
- 利用 LLM 的注意力权重识别对话历史中的关键词
- 无需额外计算开销即可显著提升查询重构质量
- 混合策略在 MRR、Recall 等指标上优于基线方法

**LDHW 方法**：
- 通过训练轻量级权重预测器，为每个 (token, head) 对学习最优权重
- 在三个数据集上 Recall@5 平均提升 4.1 个百分点
- 额外开销很小（1.2M 参数，5ms 延迟）

可解释性分析揭示了 LLM 内部的工作机制，为后续研究提供了洞见 [1][37]。
