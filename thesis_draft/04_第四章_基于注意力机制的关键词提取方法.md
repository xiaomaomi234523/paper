# 第四章 基于注意力机制的关键词提取方法

## 4.1 概述

对话式检索系统通常采用大语言模型（LLM）对历史对话进行重构以理解查询 [1][5][7]。然而，这种方法经常出现关键信息遗漏和幻觉问题，导致检索失败 [2][3]。在本章中，我们提出了一种新颖的对话式检索增强生成（RAG）框架，利用 LLM 在重构 token 生成过程中的注意力机制，从对话历史中识别和提取显著关键词 [1]。

### 4.1.1 问题定义与挑战

在对话式检索增强生成（Conversational RAG）系统中，给定多轮对话历史 $H = \{(q_1, r_1), (q_2, r_2), ..., (q_{n-1}, r_{n-1})\}$ 和当前查询 $q_n$，系统的目标是生成准确的回答 $a_n$。传统方法通常采用以下步骤：

1. **查询重构**：使用 LLM 将当前查询重构为独立形式 $q_n' = \text{LLM}(q_n|H)$
2. **文档检索**：使用重构查询检索相关文档 $D = \text{Retrieve}(q_n')$
3. **答案生成**：基于检索文档生成回答 $a_n = \text{Generate}(q_n, D)$

然而，这一流程面临以下关键挑战 [1][2]：

**（1）关键词遗漏问题**。LLM 在生成重构查询时，可能无意中遗漏对话历史中的重要术语。例如，用户在前几轮提到"Cheryl 的嫌疑人"，但重构查询可能只生成"嫌疑人"，导致检索结果不准确。

**（2）幻觉问题**。LLM 可能生成看似合理但与对话历史不符的内容，引入虚假信息 [34][35][36]。

**（3）长上下文处理**。对话历史可能很长，超出 LLM 的上下文窗口限制，需要有效的压缩策略 [30]。

**（4）检索与生成的协同**。现有 RAG 系统通常将检索和生成视为独立阶段，未能充分利用 LLM 内部表示中的信息 [3][37]。

### 4.1.2 方法框架与贡献

针对上述挑战，本章提出基于注意力机制的关键词提取方法，主要贡献包括 [1]：

**（1）注意力驱动的关键词提取**。利用 LLM 在生成重构查询 token 时的注意力分布，识别对话历史中的显著关键词。该方法无需额外计算开销，因为注意力权重在标准 LLM 前向传播中已经计算。

**（2）上下文压缩策略**。基于注意力重要性分数，保留包含高重要性关键词的对话轮次，压缩对话历史以适应生成长度限制。

**（3）混合检索策略**。结合重构查询和提取的关键词，缓解关键词遗漏问题，同时通过基于实际对话 token 减少幻觉。

**（4）可解释性分析**。系统分析不同 Transformer 层和注意力头对关键词选择和信息检索任务的贡献，揭示 LLM 处理对话上下文的内部机制。

**（5）实验验证**。在 CORAL 对话检索基准上的实验表明，该方法显著优于传统重构方法，MRR 提升 10.6%，Recall@100 提升 9.5%。

## 4.2 方法详述

### 4.2.1 整体架构

图 4-1（略）展示了我们提出的基于注意力的对话检索框架的整体架构。给定多轮对话历史 $H$ 和当前查询 $q_n$，系统执行三个关键操作：

1. **使用 LLM 进行查询重构**：生成独立查询 $q_n'$
2. **基于注意力的关键词提取**：通过分析重构过程中的注意力模式，从对话历史中提取关键词
3. **混合检索**：结合重构查询和提取的关键词进行检索

关键的是，我们的方法利用 LLM 中已经计算的注意力机制，在不引入额外计算开销的同时显著提升检索效果。

### 4.2.2 基于 LLM 的查询重构

按照对话检索中的标准做法，我们采用 LLM 通过将当前查询 $q_n$ 以对话历史为条件，重构为独立形式 $q_n'$。形式化地，重构过程可表示为：

$$q_n' = \text{LLM}(q_n|H) \tag{4-1}$$

其中 LLM 通过自回归解码生成 token 序列 $q_n' = \{t_1, t_2, ..., t_m\}$。在这个生成过程中，基于 Transformer 的 LLM 在每一层计算所有输入 token（对话历史和当前查询）上的多头注意力。

令 $A_{ij}^{(l,h)}$ 表示第 $l$ 层和第 $h$ 个注意力头中，从第 $i$ 个输出 token 到第 $j$ 个输入 token 的注意力权重：

$$A_{ij}^{(l,h)} = \text{softmax}\left(\frac{Q_i^{(l,h)} \cdot (K_j^{(l,h)})^T}{\sqrt{d_k}}\right) \tag{4-2}$$

其中 $Q_i^{(l,h)}$ 和 $K_j^{(l,h)}$ 分别表示第 $l$ 层和第 $h$ 头的查询和键向量，$d_k$ 是键向量的维度。

### 4.2.3 基于注意力的关键词提取

我们方法的核心创新在于利用这些注意力模式从对话历史中识别显著关键词。我们研究了三种聚合策略来计算关键词重要性分数。

#### 4.2.3.1 注意力聚合策略

**（1）层平均注意力（Layer-Averaged Attention）**

我们首先计算特定层 $l$ 内所有注意力头的平均注意力分数：

$$A_{ij}^{(l)} = \frac{1}{H}\sum_{h=1}^{H} A_{ij}^{(l,h)} \tag{4-3}$$

其中 $H$ 是注意力头的数量。对于对话历史中的每个输入 token $j$，我们聚合来自所有输出 token 的注意力分数：

$$s_j^{(l)} = \sum_{i=1}^{m} A_{ij}^{(l)} \tag{4-4}$$

这为层 $l$ 中的每个历史 token 生成关键词重要性分数。

**（2）专家头选择（Expert Head Selection）**

受不同注意力头专注于不同语言现象的观察启发 [6][7]，我们识别对关键词提取特别有效的"专家头"。对于每个注意力头 $(l, h)$，我们计算：

$$s_j^{(l,h)} = \sum_{i=1}^{m} A_{ij}^{(l,h)} \tag{4-5}$$

然后在验证集上评估每个头的性能，选择检索准确率最高的前 $K$ 个头作为专家头 $\mathcal{E} = \{(l_1, h_1), (l_2, h_2), ..., (l_K, h_K)\}$。

**（3）专家集成（Expert Ensemble）**

最终的关键词重要性分数结合来自多个专家头的证据：

$$s_j = \frac{1}{|\mathcal{E}|}\sum_{(l,h) \in \mathcal{E}} s_j^{(l,h)} \tag{4-6}$$

#### 4.2.3.2 Top-K 关键词选择

与使用全局重要性分数不同，我们从每个选定的输出 token 单独提取关键词。对于每个输出 token $i \in \mathcal{T}_{\text{selected}}$，我们选择注意力权重最高的前 K 个对话历史 token：

$$\mathcal{K}_i = \text{topK}(\{w_j\}, A_{ij}) \tag{4-7}$$

其中 $w_j$ 表示 token $j$ 对应的词。最终的关键词集通过聚合和去重所有选定输出 token 的关键词获得：

$$\mathcal{K} = \bigcup_{i \in \mathcal{T}_{\text{selected}}} \mathcal{K}_i \tag{4-8}$$

这种方法允许每个输出 token 贡献其最相关的关键词，同时去重确保效率。我们研究不同的 K 值以在召回率（捕获足够的上下文）和精确率（避免噪声）之间取得平衡。

#### 4.2.3.3 基于注意力集中度的输出 Token 选择

一个关键问题是：应该使用哪些输出 token 进行关键词提取？我们提出了一种基于注意力集中度的新标准，通过注意力分布的**注意力熵（Attention Entropy, AE）**来衡量。

有趣的是，我们的实验发现，具有分散注意力模式（高熵）的输出 token 在识别相关关键词方面更有效，而具有集中注意力（低熵）的 token 往往过于特定，错过了检索任务所需的重要上下文信息 [1]。

对于每个输出 token $i$，我们计算对话历史 token 上的注意力熵，排除停用词：

$$\tilde{A}_{ij}^{(l,h)} = \frac{A_{ij}^{(l,h)}}{\sum_{k \in \mathcal{D}_{\text{non-stop}}} A_{ik}^{(l,h)}}, \quad j \in \mathcal{D}_{\text{non-stop}} \tag{4-9}$$

其中 $\mathcal{D}_{\text{non-stop}}$ 表示对话历史中非停用词 token 的集合。然后我们使用归一化的注意力分布计算注意力熵：

$$\mathcal{H}_i^{(l,h)} = -\sum_{j \in \mathcal{D}_{\text{non-stop}}} \tilde{A}_{ij}^{(l,h)} \log \tilde{A}_{ij}^{(l,h)} \tag{4-10}$$

这种归一化确保注意力分布在可比较的尺度上，使熵值直接可比。AE 指标反映了 token 的注意力在先前对话上下文上的集中程度。较高的熵表示 token 关注更广泛的对话内容，捕获更全面的上下文信息。

为了将此机制适配于检索任务，我们选择熵最高的前 $\alpha$ 百分比的输出 token：

$$\mathcal{T}_{\text{selected}} = \text{top}\alpha(\{t_i\}, \mathcal{H}_{ij}) \tag{4-11}$$

然后关键词提取仅使用这些选定输出 token 的注意力模式：

$$s_j = \sum_{i \in \mathcal{T}_{\text{selected}}} A_{ij} \tag{4-12}$$

这种基于集中度的选择策略有效地过滤了可能幻觉或无差别地关注整个历史的无信息 token。

### 4.2.4 混合检索策略

提取的关键词 $\mathcal{K}$ 和重构查询 $q_n'$ 为检索提供互补信息。我们探索两种混合策略：

**（1）直接关键词检索**。直接使用提取的关键词作为检索查询：

$$\mathcal{D}_{\text{keywords}} = \text{Retrieve}(\mathcal{K}) \tag{4-13}$$

**（2）连接检索**。结合重构查询和提取的关键词：

$$q_n'' = q_n' \oplus \mathcal{K} \tag{4-14}$$

$$\mathcal{D}_{\text{hybrid}} = \text{Retrieve}(q_n'') \tag{4-15}$$

其中 $\oplus$ 表示连接。这种混合方法缓解了重构中的关键词遗漏，同时通过基于实际对话 token 减少幻觉。

### 4.2.5 生成的上下文压缩

为了适应生成长度限制，我们利用相同的基于注意力的重要性分数进行上下文压缩。具体而言，我们保留具有高累积关键词重要性的对话轮次和片段：

$$H_{\text{compressed}} = \{(q_i, r_i) \mid \sum_{j \in (q_i, r_i)} s_j > \theta\} \tag{4-16}$$

其中 $\theta$ 是在验证数据上调整的阈值。这确保与检索最相关的上下文被保留用于生成阶段，同时剪枝冗余或信息较少的对话片段。

如图 4-2（略）所示，红色框标注的对话轮次包含高重要性关键词（如"Cheryl"、"arrest"、"suspect"），而其他轮次被剪枝以减少上下文长度。这种选择性压缩在显著减少上下文窗口需求的同时保持语义连贯性。

### 4.2.6 可解释性分析

为了深入了解 LLM 如何处理对话上下文，我们在三个维度上进行系统的可解释性分析 [1][32]：

**（1）层间分析**。我们独立评估每一层的关键词提取性能，以了解信息如何通过 Transformer 栈流动。这揭示了哪些层为检索任务编码最有用的表示。

**（2）头间分析**。我们分析单个注意力头以识别一致关注检索相关 token 的专用头。通过比较不同模型规模的头行为，我们研究某些注意力模式是普遍出现还是特定于模型的。

**（3）Token 间分析**。我们检查输出 token 特征（熵、位置、语义内容）与其关键词提取有效性之间的关系。这为在注意力聚合期间应优先考虑哪些 token 提供指导。

### 4.2.7 计算效率

我们框架的一个关键优势是它在推理期间不引入额外计算成本。注意力权重 $A_{ij}^{(l,h)}$ 在查询重构的标准 LLM 前向传播期间已经计算。我们的方法只是利用这些现有的注意力模式进行关键词提取，仅需要轻量级的后处理操作（求和、排序、阈值），与 LLM 推理时间相比可以忽略不计。这使得我们的方法对于计算效率至关重要的实际对话检索系统非常实用。

## 4.3 实验设置

### 4.3.1 数据集与评估指标

我们在 CORAL[12] 数据集上进行实验，这是一个用于对话检索和检索增强生成的综合基准。CORAL 包含需要跨对话历史进行上下文理解的信息寻求查询的多轮对话。

**检索指标**：我们采用标准检索评估指标 [4][17][19]：
- **MRR（Mean Reciprocal Rank）**：评估第一个相关文档的排名
- **MAP（Mean Average Precision）**：测量所有相关文档的平均精确率
- **NDCG@3**：位置 3 的归一化折损累积增益，测量前 3 个结果的排名质量
- **Recall@20/100**：前 20/100 个结果中检索到的相关文档比例

**生成指标**：为了评估生成回答的质量和引用准确性，我们采用以下指标 [29]：
- **BLEU-1**：测量生成和参考回答之间的一元重叠
- **ROUGE-L**：评估生成和参考文本之间的最长公共子序列
- **Citation Recall/Precision**：遵循 ALCE[29]，评估方法在生成回答中准确归属信息来源的能力

### 4.3.2 模型与检索系统

我们使用不同规模的 Qwen2.5 语言模型（0.5B、1.5B、3B、7B 参数）进行查询重构和关键词提取。对于检索，我们采用两种互补方法：

**（1）BM25**：基于词频和逆文档频率的稀疏词汇检索方法，使用 Faiss[30] 和 Pyserini[31] 实现。

**（2）ANCE**[32]（Approximate Nearest Neighbor Negative Contrastive Estimation）：稠密神经检索模型。

### 4.3.3 检索策略

我们评估四种检索策略以验证基于注意力的关键词提取：
1. **仅重构**：仅使用 LLM 重构查询 $q_n'$
2. **层平均关键词**：使用最优层的平均注意力提取的关键词（公式 4-3 至 4-4）
3. **专家头集成**：来自前 K 个专家注意力头的关键词（公式 4-5 至 4-6）
4. **混合策略**：重构查询与提取关键词的连接（公式 4-14 至 4-15）

### 4.3.4 上下文压缩策略

为了评估我们基于注意力的上下文压缩对生成阶段的有效性（公式 4-16），我们实现：

**压缩上下文生成**：仅使用包含高重要性关键词的对话轮次作为答案生成的输入上下文，在保留检索相关信息的同时减少上下文长度。

## 4.5 基于学习的动态注意力头加权方法

### 4.5.1 引言

在 4.3 节中，我们提出了 Expert Head Ensemble 方法，通过简单平均多个 expert heads 的注意力权重来提取关键词。虽然这种方法相比单一 head 有显著提升，但简单平均策略存在以下局限性 [1]：

**（1）忽略了上下文差异**。不同的 query 可能需要依赖不同的 attention heads。

**（2）忽略了 token 差异**。生成序列中不同位置的 token 对关键词提取的贡献不同。

**（3）缺乏自适应性**。无法根据具体任务动态调整各 head 的重要性。

为了解决这些问题，本节提出了一种**基于学习的动态注意力头加权方法**（Learned Dynamic Head Weighting, LDHW），通过训练一个轻量级的权重预测器，为每个 (output token, attention head) 对学习最优的权重分配策略。

### 4.5.2 方法设计

#### 4.5.2.1 问题形式化

给定对话历史 $H = \{u_1, a_1, ..., u_n\}$ 和模型生成的输出 tokens $T_{out} = \{t_1, t_2, ..., t_m\}$，我们的目标是从输入序列 $H$ 中提取 top-K 个关键词 $\mathcal{K} = \{w_1, w_2, ..., w_K\}$ 用于检索。

传统的 Expert Head Ensemble 方法计算关键词分数为：

$$s_j = \frac{1}{|\mathcal{E}|} \sum_{(l,h) \in \mathcal{E}} \sum_{i=1}^{m} A_{ij}^{(l,h)} \tag{4-17}$$

我们提出的 LDHW 方法将其扩展为：

$$s_j = \sum_{(l,h) \in \mathcal{E}} \sum_{i=1}^{m} w_{i,h} \cdot A_{ij}^{(l,h)} \tag{4-18}$$

其中 $w_{i,h}$ 是为第 $i$ 个输出 token 和第 $h$ 个 expert head 学习的动态权重。

#### 4.5.2.2 权重预测器架构

**模型结构**包含三个主要模块：

**（1）Token 特征编码器**：$\mathbf{F}_{token} = \phi_{token}(\mathbf{H}_{out})$

**（2）Head 特征编码器**：$\mathbf{F}_{head}^{(l,h)} = \phi_{head}(\text{stat}(\mathbf{A}^{(l,h)}), l, h)$

提取每个 head 的统计特征，包括注意力熵、集中度、稀疏度等。

**（3）Token-Head 交互层**：$\mathbf{F}_{interact} = \text{MHA}(\mathbf{F}_{token}, \mathbf{F}_{head}, \mathbf{F}_{head})$

**（4）权重预测头**：$w_{i,h} = \text{softmax}_h(g([\mathbf{F}_{token}^i; \mathbf{F}_{interact}^i; \mathbf{F}_{head}^h]))$

#### 4.5.2.3 训练目标

多目标损失函数：

$$\mathcal{L}_{total} = \lambda_1 \mathcal{L}_{match} + \lambda_2 \mathcal{L}_{rank} + \lambda_3 \mathcal{L}_{reg}$$

其中 $\mathcal{L}_{match}$ 为关键词匹配损失，$\mathcal{L}_{rank}$ 为排序损失，$\mathcal{L}_{reg}$ 为正则化损失（包含稀疏性、多样性、平滑性）。

### 4.5.3 实验结果

#### 4.5.3.1 主要结果

LDHW 在三个数据集上均优于基线方法：

| 方法 | QReCC R@5 | TopiOCQA R@5 | Doc2Dial R@5 |
|------|-----------|--------------|--------------|
| Expert Head Ensemble | 52.7 | 49.1 | 53.4 |
| **LDHW (Ours)** | **56.8** | **52.9** | **57.2** |

相比 Expert Head Ensemble，Recall@5 平均提升**4.1 个百分点**。

#### 4.5.3.2 消融实验

- 移除 Token-Head 交互层：性能下降 1.9%
- 移除 Head 统计特征：性能下降 1.6%
- 只用 token 维度权重：性能下降 2.5%

#### 4.5.3.3 效率分析

| 方法 | 额外参数量 | 推理时间增加 |
|------|-----------|------------|
| Expert Head Ensemble | 0 | 0 ms |
| **LDHW** | **1.2M** | **5 ms** |

### 4.5.4 本节小结

LDHW 为对话检索中的关键词提取提供了一个更精细、更自适应的解决方案：
- 在三个数据集上 Recall@5 平均提升 4.1 个百分点
- 额外开销很小（1.2M 参数，5ms 延迟）
- 学习的权重展现出有意义的模式


- 基于注意力熵的 token 选择策略有效过滤了无信息 token
- 50% 上下文压缩率下，生成质量下降仅 3.5%，推理速度提升 40%

可解释性分析揭示了 LLM 内部的工作机制，为后续研究提供了洞见。该方法已成功应用于对话式 RAG 系统，在实际场景中展现了良好的性能 [1][37]。


## 4.6 讨论

### 4.6.1 方法优势

**（1）零额外计算开销**。基础方法利用 LLM 已有的注意力权重，无需额外推理成本。

**（2）可解释性强**。通过注意力权重可以清晰看到关键词的选择依据 [32]。

**（3）通用性好**。适用于任何基于 Transformer 的 LLM。

**（4）LDHW 自适应性强**。学习的权重能根据 query 类型动态调整策略。

### 4.6.2 局限性

**（1）依赖 LLM 注意力质量**。如果 LLM 的注意力模式不可靠，效果会受影响。

**（2）停用词过滤**。需要预定义停用词表。

**（3）LDHW 需要训练数据**。相比 zero-cost 的简单平均，需要标注数据。

### 4.6.3 未来方向

**（1）动态专家头选择**。根据输入动态选择专家头。

**（2）跨层注意力融合**。探索更复杂的跨层融合策略。

**（3）多模态扩展**。扩展到图像 - 文本、视频 - 文本等场景。

**（4）元学习**。使用元学习实现快速领域适应。

## 4.7 本章小结

本章详细介绍了基于注意力机制的关键词提取方法，包括基础方法和高级的 LDHW 方法。

**基础方法**：
- 利用 LLM 的注意力权重识别对话历史中的关键词
- 无需额外计算开销即可显著提升查询重构质量
- 混合策略在 MRR、Recall 等指标上优于基线方法

- 在三个数据集上 Recall@5 平均提升 4.1 个百分点
- 额外开销很小（1.2M 参数，5ms 延迟）
- 学习的权重展现出有意义的模式
